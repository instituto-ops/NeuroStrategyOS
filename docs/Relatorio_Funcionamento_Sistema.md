# Relatório de Funcionamento: AntiGravity CMS - Mission Control

## 1. Visão Geral
O AntiGravity CMS é uma plataforma de gerenciamento de conteúdo headless projetada para orquestrar a presença digital do Instituto Lawrence de Hipnose Clínica. Diferente de painéis WordPress convencionais, este sistema opera de forma desacoplada, utilizando Inteligência Artificial de ponta (Google Gemini) para garantir que toda comunicação adira aos padrões científicos e éticos (E-E-A-T) do Dr. Victor Lawrence.

## 2. Arquitetura Técnica
O sistema utiliza uma arquitetura unificada e moderna, focada em performance e segurança:

*   **Motor (Servidor Unificado):** Baseado em Node.js (Express), rodando na porta 3000. Ele serve tanto a interface visual quanto a camada de API.
*   **WordPress Headless:** O WordPress é utilizado puramente como um banco de dados de conteúdo. A comunicação ocorre via REST API, permitindo que o frontend seja extremamente rápido e livre das limitações visuais dos temas tradicionais.
*   **Proxy de Segurança:** Todas as requisições ao WordPress passam por um proxy interno. Isso evita a exposição de credenciais no navegador e contorna bloqueios comuns de servidores (como o Erro 403 do ModSecurity na Hostinger).

## 3. Módulos de Inteligência e Automação

### A. Módulo Neuro-Engine (Clone de Voz)
O sistema possui uma memória de estilo baseada no DNA de escrita do Dr. Victor Lawrence.
*   **Reverse Prompting:** A IA analisa textos autênticos para aprender cadência, vocabulário técnico (TEA, Psicometria, Hipnose Ericksoniana) e termos proibidos.
*   **Perfil de Síntese Científica:** Injeta automaticamente dados de autoridade (CRP, mestrado na UFU, ORCID) em cada rascunho gerado para garantir conformidade acadêmica e SEO.

### B. Sistema Abidos (Design & Auditoria)
O "Método Abidos" é a espinha dorsal visual e de qualidade do sistema:
*   **Abidos Prestige (Template):** Um sistema de design único que utiliza CSS moderno (Glassmorphism, Micro-animações, Container Queries) sem depender de bibliotecas pesadas.
*   **Agente Abidos (Auditor):** Antes da publicação, um subagente de IA realiza uma auditoria clínica no texto (FactScore), verificando riscos éticos (CFP/LGPD) e alucinações técnicas.

### C. Mission Control Pipeline
O fluxo de trabalho segue a filosofia Agent-First:
1.  **Geração:** O "Conselho de Agentes" cria um rascunho com base em diretrizes de estilo e SEO local (Goiânia).
2.  **Refino:** O sistema analisa a diferença (diff) entre o rascunho da IA e as edições manuais do usuário para aprender e não repetir "erros" de tom.
3.  **Publicação:** Após aprovação, o conteúdo é enviado para o WordPress já formatado para o Elementor ou Gutenberg.

## 4. Integrações Especiais
*   **Doctoralia Sync:** Sincronização e análise de avaliações reais para extrair insights estratégicos e provas sociais autênticas.
*   **SEO Programático (Silos):** Organização automática do conteúdo em silos de autoridade (Ansiedade, Autismo, Depressão), garantindo que o Google entenda a hierarquia diagnóstica do site.

## 5. Resumo da Stack Tecnológica
- Backend/API: Node.js, Express, Axios
- Inteligência Artificial: Gemini Flash (Padrão): Para navegação, extrações rápidas e SEO (velocidade e custo). Gemini Pro: Ativado automaticamente para "Orquestrações Abidos" e análises clínicas complexas (profundidade).
- Frontend: Vanilla JavaScript, CSS3 Contemporâneo, Tailwind CSS
- Banco de Dados: WordPress REST API + MariaDB (Remoto)
- Processamento: Multer (Arquivos), PDF-Parse, Mammoth (Docx)
