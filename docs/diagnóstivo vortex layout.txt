## A causa raiz dos bugs comportamentais

**Auto-navegação para Etapa 2.** Quando você clica num silo, o estado interno (`vortexStudio.openVisualSilo`) chama `setVisualLayer('edit')` no fim. Esse comportamento foi pensado para "editar página existente" mas está atrelado a qualquer interação com silo, inclusive seleção contextual. Precisa ter dois modos distintos: *referenciar silo no briefing* (não navega) vs *abrir spoke para editar* (navega). Hoje só existe o segundo.

**"Adicionar Spoke" não funciona.** O botão `+ Novo Spoke` chama `startNewSpokeFromSilo()` mas o método não está exportado no `return` do módulo `vortexStudio`. Mesmo problema dos outros handlers que o Codex deixou sem exportar — o Claude Code repetiu o erro. É uma linha de código faltando.

**Keyword Tier 1 hardcoded.** A lista vem de uma constante `TIER_1_KEYWORDS` definida no JS. Para virar editável precisa virar um array em `state.creationBrief.keywordsCatalog` com input livre + persist no localStorage. Sem botão para adicionar nova, o sistema é decorativo.

**Botões "Criar" e "Editar" do Briefing.** Pelo que vejo no código, eles deveriam alternar `state.creationBrief.mode` entre `create` e `edit`. Mas como esse modo não muda nada visualmente em lugar nenhum, o usuário não sabe o que estão criando ou editando. **Devem ser removidos.** A lógica certa é: o briefing **é** o modo Criar; abrir um spoke **é** o modo Editar. Os dois botões viraram artefato sem propósito.

**Silo aparece duplicado (no topo + no campo do briefing).** O dropdown "HUB / SILO" no formulário de briefing é redundante com o painel de silos acima. Mantém só um — o painel visual com silos colapsáveis. O dropdown some.

**Preview espremido.** O grid CSS reserva 283px para um painel oculto (chat/editor do V5 antigo). Esse painel precisa virar `display: none` quando `data-vortex-mode="visual"`. Se ele simplesmente sumir, o preview ganha 283px de altura imediatamente. Foi exatamente o bug que apontei vários turnos atrás e que o Claude Code não atacou.

**Falta toggle viewport.** Não existe na implementação. Precisa entrar como controle persistente acima do preview, não como botão "Otimizar Mobile" no dock (que é outra coisa: reescreve o conteúdo, não muda preview).

**Falta edição manual `contenteditable`.** Idem — está no plano mas o Claude Code não implementou. O preview hoje é iframe estático com `pointer-events` que abre um campo de prompt — não permite digitar no texto.

**Falta botão de cancelar seleção.** Quando você clica num campo, vira "campo selecionado" sem saída. Precisa de um `Esc` ou X visível.

**Conteúdo "masking" persiste apesar de você pedir hipnose clínica.** O preview está renderizando um `template-values.json` estático ou um draft em cache, não regenerando do briefing. Quando você muda de keyword/silo, o preview tinha que ser invalidado (ou mostrar placeholder "Pronto para gerar — clique em Gerar com Briefing").

**Mojibake nos templates.** Continua. O Claude Code também não tocou no JSON do catálogo.

✅ 7 PADRÕES A COPIAR DO ESTÚDIO PARA O VORTEX
1. Grid de 3 Colunas Fixas e Simétrico
Estúdio mantém sempre 3 cards paralelos (L | C | R) em todas as etapas. Vortex deveria fazer o mesmo: grid-template-columns: 1fr 2fr 1fr com larguras proporcionais e sem overlap.
2. Abas Horizontais no Topo do Painel Esquerdo
Em vez de hierar quia de painel (Silos escondidos, Briefing rolável, Alterações sobrepostas), usar abas tipo:

SILOS | BRIEFING | EDIÇÕES
Cada etapa ativa sua aba correspondente automaticamente.

3. Padrão de Card Padding e Espaçamento
Todos os cards do Estúdio têm padding: 24px, border-radius: 12px, bordas sutis e gap: 20px. Vortex tem cards, mas sem consistência — padronizar isso.
4. Preview com 100% de Altura e Scroll Interno
Estúdio mostra "Preview Real-time" com altura máxima e scroll. Vortex trava em 433px com overflow hidden. Mudar para height: calc(100vh - XXpx) com overflow-y: auto.
5. Botões de Ação em Zona Dedicada
Estúdio coloca os botões GERAR e AUDITAR numa faixa clara. Vortex distribui ações (Refinar Hero, etc.) flutuando sobre o preview. Criar uma barra de ações acima do preview com: display: grid | gap: 12px | justify-items: start.
6. Indicadores de Status Claros
Estúdio exibe status real: "Aguardando Geração" ou "Preview Amplo". Vortex não dá feedback. Adicionar: labels tipo "🟢 Pronto | 🟡 Processando | 🔴 Erro".
7. Breadcrumb / Localização no Cabeçalho
Estúdio inicia com "Estúdio de Conteúdo" + número e nome da etapa. Vortex coloca os números soltos no meio. Mover a barra de etapas para a navbar ou criar um breadcrumb dedicado tipo: "Vortex · Sinais Sutis em Mulheres · Etapa 2: Edição".

✅ Ações em barra dedicada (não flutuante)
✅ Status visual imediato
✅ Espaço sempre aproveitado (sem áreas pretas vazias)
✅ Card design consistente (padding, gap, radius)
