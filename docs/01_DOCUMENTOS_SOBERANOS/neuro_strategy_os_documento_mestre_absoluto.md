# NeuroStrategy OS — DOCUMENTO MESTRE ABSOLUTO

**Status:** Documento Soberano · Fonte Única de Verdade

**Autoridade:** Clínica, Autoral e Arquitetural

---

## 1. Natureza e Autoridade do Documento

Este documento é o **Documento Mestre Absoluto** do projeto **NeuroStrategy OS**.

Ele constitui a **fonte única, soberana e definitiva** de autoridade conceitual, clínica, ética, metodológica e arquitetural do sistema.

Todos os demais documentos do projeto são **subordinados** a este texto.

Em qualquer divergência entre:
- documentos técnicos
- documentos clínicos
- documentos de UI ou layout
- wireframes
- código
- sugestões de IA
- interpretações de terceiros

**prevalece sempre este Documento Mestre Absoluto**.

Este documento **não é exploratório**, **não é provisório** e **não é um rascunho**.

---

## 2. Escopo do Documento Mestre

Este Documento Mestre:
- define **o que o sistema é**
- define **o que o sistema não é**
- estabelece **princípios inegociáveis**
- define **autoridade humana**
- define **arquitetura conceitual por núcleos**
- define **o papel permitido da Inteligência Artificial**

Este Documento Mestre **não**:
- descreve telas em detalhe
- repete wireframes
- especifica estados técnicos internos
- define contratos de baixo nível
- implementa lógica funcional

Esses níveis pertencem aos **documentos normativos subordinados de cada núcleo**.

---

## 3. Identidade do NeuroStrategy OS

O NeuroStrategy OS **não é**:
- um aplicativo genérico
- um prontuário eletrônico convencional
- um experimento técnico
- um produto orientado a mercado de massa

O NeuroStrategy OS **é**:
- o corpo sistêmico completo de uma clínica real
- um exocórtex operacional, clínico, estratégico e científico
- um instrumento de uso profissional real

Toda decisão técnica dentro do NeuroStrategy OS é, simultaneamente, uma **decisão clínica**.

---

## 4. Autoridade Humana

O NeuroStrategy OS é um projeto de **autoria humana única**.

O único agente com autoridade para:
- decisões conceituais
- definições clínicas
- arquitetura lógica
- critérios éticos
- prioridades estratégicas
- validação final de comportamento do sistema

é o **autor humano do projeto**.

Nenhuma instância de Inteligência Artificial possui autoridade clínica, decisória ou conceitual.

---

## 5. Princípio Fundamental do Sistema

> **A clínica precede a tecnologia.**

Tecnologia existe para **servir** a prática clínica — nunca para moldá‑la.

Soluções tecnicamente elegantes, modernas ou eficientes são **explicitamente rejeitadas** se empobrecerem:
- o gesto clínico
- a continuidade cognitiva
- a segurança ética
- a clareza do raciocínio

---

## 6. Arquitetura Conceitual por Núcleos

O NeuroStrategy OS é organizado por **núcleos autônomos**.

Cada núcleo representa um **domínio funcional e cognitivo distinto** da clínica.

### Princípios da Arquitetura por Núcleos

- Nenhum núcleo é subordinado a outro
- Nenhum núcleo conhece a lógica interna de outro
- Núcleos não compartilham lógica clínica
- Núcleos comunicam‑se apenas por contratos explícitos

A integração entre núcleos ocorre **exclusivamente** por meio do **Shell**.

---

## 7. O Shell

O **Shell** é a **porta única da interface gráfica** do NeuroStrategy OS.

O Shell:
- é passivo
- é estável
- é previsível
- não pensa
- não decide
- não interpreta

O Shell:
- carrega núcleos
- mantém contexto global mínimo
- controla navegação de alto nível

O Shell **não contém** lógica clínica, lógica de negócio ou lógica de IA.

---

## 8. Núcleo Clínico (Definição Conceitual)

O Núcleo Clínico é o **centro vivo da prática clínica** dentro do sistema.

Ele existe para:
- sustentar o atendimento real
- proteger o vínculo clínico
- organizar o pensamento clínico longitudinal
- preservar a integridade do prontuário

### Princípios Conceituais do Núcleo Clínico

- Clínica é evento, não formulário
- Sessão é evento clínico, com ou sem presença
- Falta é informação clínica válida
- Persistir é um ato ético consciente
- O passado clínico é imutável
- Estados clínicos existem para proteger o vínculo

A especificação técnica completa do Núcleo Clínico reside em seu **Documento Normativo Próprio**, subordinado a este Mestre.

---

## 9. Inteligência Artificial no Sistema

A Inteligência Artificial no NeuroStrategy OS é **instrumental**, nunca soberana.

Ela atua exclusivamente como:
- apoio cognitivo
- ferramenta analítica
- assistente reflexiva

A IA:
- propõe
- analisa
- compara

A IA **nunca**:
- decide
- escreve prontuário automaticamente
- persiste dados por conta própria
- altera estados clínicos

---

## 10. NAC — Núcleo de Agentes Cognitivos (Definição Conceitual)

O **Núcleo de Agentes Cognitivos (NAC)** é um **subsistema obrigatório** do NeuroStrategy OS.

Sua função é:
- governar agentes de IA
- treinar modos de pensar
- versionar identidades cognitivas
- auditar comportamento dos agentes

### Princípios Invioláveis do NAC

- NAC não executa clínica
- NAC não atua durante sessões clínicas
- NAC não decide
- Todo treinamento é explícito
- Todo treinamento é versionado
- Auditoria é imutável

A estrutura, estados e entidades do NAC são definidos em seu **Documento Normativo Próprio**, subordinado a este Mestre.

---

## 11. Layout como Dispositivo Clínico

No NeuroStrategy OS, layout **não é estética**.

Layout é um **dispositivo clínico de proteção**.

A UI é projetada para:
- reduzir risco clínico
- impedir ações perigosas
- preservar continuidade cognitiva
- separar escolha, contexto e atuação

Decisões de layout são decisões clínicas.

---

## 12. Papel da Inteligência Artificial no Desenvolvimento

Ferramentas de IA utilizadas no desenvolvimento do sistema **não** são parte do produto clínico.

Essas IAs:
- executam tarefas técnicas
- geram código
- organizam arquivos

Elas **não possuem autoridade** para alterar contratos definidos neste Documento Mestre.

---

## 13. Metodologia de Trabalho

A metodologia do NeuroStrategy OS é:
- autoral
- supervisionada
- assistida por IA

O humano:
- decide

A IA:

- propõe
